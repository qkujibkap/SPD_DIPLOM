DATA_SCHEMA — структура файла favorites.json

Файл favorites.json используется для хранения списка избранных пользователей, выбранных через бота VKinder.

Каждый элемент списка имеет следующую структуру:

    {
      "id": 123456789,
      "first_name": "Иван",
      "last_name": "Иванов",
      "profile_url": "https://vk.com/id123456789",
      "photos": [
        "photo123456789_987654321",
        "photo123456789_123400000",
        "photo123456789_543210000"
      ],
      "_added_at": "2025-01-21T19:26:03Z"
    }

------------------------------------------------------------
Описание полей
------------------------------------------------------------

id (integer)
    Уникальный идентификатор пользователя ВКонтакте.

first_name (string)
    Имя пользователя.

last_name (string)
    Фамилия пользователя.

profile_url (string)
    Прямая ссылка на профиль формата https://vk.com/id{id}

photos (array of strings)
    Список фото вида "photo{owner_id}_{photo_id}".
    Используется как вложения в vk.messages.send.

_added_at (string, ISO-8601)
    Дата и время добавления записи.
    Пример: 2025-01-21T19:26:03Z

------------------------------------------------------------
Пример favorites.json
------------------------------------------------------------

    [
      {
        "id": 123,
        "first_name": "Анна",
        "last_name": "Петрова",
        "profile_url": "https://vk.com/id123",
        "photos": [
          "photo123_111",
          "photo123_222",
          "photo123_333"
        ],
        "_added_at": "2025-01-22T10:15:12Z"
      }
    ]

------------------------------------------------------------
Примечания
------------------------------------------------------------

• Файл создаётся автоматически при первом запуске, если отсутствует.

• При повреждении структуры (битый JSON, неверный формат) данные будут сброшены в пустой список.

• Повторное добавление пользователя с тем же id не допускается.
